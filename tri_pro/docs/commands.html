▶️ 일상 운영

3개 서버(backend 8000 / patient 3000 / admin 3001) 시작·종료·재시작.

서비스 시작 (idempotent)로컬
이미 떠있으면 그대로, 안 떠있는 것만 띄움. 가장 자주 쓰는 명령.
./start.sh
포그라운드 시작 (로그 한 화면에)로컬
3개 서버 로그를 [backend][patient][admin] 컬러로 합쳐서 출력. Ctrl+C로 한 번에 종료.
./start.sh --fg
상태 확인
3개 서버 ●up / ○down 표시.
./restart.sh status
로그 보기 (백그라운드 시작했을 때)
.logs/{backend,patient,admin}.log tail.
./restart.sh logs
전체 종료
./stop.sh
강제 재시작 (살아있어도 kill 후 새로)
./restart.sh
DB 리셋 + 재시작데이터 삭제
스키마 변경 후 적용 시 사용. 응답·구독 모두 초기화.
./restart.sh --reset
.next 캐시 삭제 + 재시작
Next.js 라우트가 404로 나오거나 변경분이 안 잡힐 때.
./restart.sh --clean
전부 (DB + 캐시) 리셋 후 재시작데이터 삭제
./restart.sh --all

🔍 트러블슈팅

포트 점유·헬스체크·로그·스키마 충돌 해결.

포트 점유 확인
lsof -i tcp:8000 lsof -i tcp:3000 lsof -i tcp:3001
포트 강제 종료
lsof -ti tcp:8000 | xargs kill -9 lsof -ti tcp:3000 | xargs kill -9 lsof -ti tcp:3001 | xargs kill -9
서버 헬스체크
curl -s http://localhost:8000/api/health curl -s -o /dev/null -w "%{http_code}\n" http://localhost:3000 curl -s -o /dev/null -w "%{http_code}\n" http://localhost:3001
신규 라우트 5종 검증
for p in \ "/api/health" \ "/api/patients/C-1042/pro-config" \ "/api/pro-tools/library" \ "/api/pro-tools/recommend?icd10=C18" \ "/api/patients/C-1042/full-mode-status"; do printf "%-50s " "$p" curl -sf -o /dev/null -w "%{http_code}\n" "http://localhost:8000$p" done
대시보드 환자 5명 응답 확인
curl -s http://localhost:8000/api/admin/dashboard | python3 -m json.tool
Web Push 구독 디바이스 확인
curl -s http://localhost:8000/api/push/subscriptions/C-1042 | python3 -m json.tool
백엔드 로그 마지막 50줄
tail -50 .logs/backend.log
백엔드 로그 실시간 follow
tail -f .logs/backend.log
.next 캐시 완전 삭제 (404 잘못 뜰 때)
rm -rf admin-app/.next patient-app/.next ./restart.sh
Python 캐시 삭제 (stale .pyc 의심 시)
find . -type d -name __pycache__ -not -path "./*/venv/*" -exec rm -rf {} + find . -name "*.pyc" -delete
DB 스키마 충돌 (테이블 컬럼 추가했는데 안 잡힐 때)
init_db()는 컬럼 마이그레이션 안 함. dev.db 통째 새로 만들어야.
./restart.sh --reset

☁️ EC2 배포

단일 서버 + Caddy 자동 HTTPS + 5분 만에 다른 사람과 공유.

EC2 첫 셋업 (1회)EC2
Docker·docker compose plugin 설치. ssh 접속 후 실행.
ssh -i ~/.ssh/tri-pro.pem ec2-user@<EIP> git clone https://github.com/aitrics/tri-pro-poc.git cd tri-pro-poc bash scripts/ec2_setup.sh exit # docker 그룹 적용 위해 재로그인
.env.prod 작성EC2
cp .env.prod.example .env python3 backend/scripts/gen_vapid.py # 출력값을 .env에 복사 vim .env
첫 배포EC2
./scripts/deploy.sh
재배포 (코드 푸시 후)EC2
로컬에서 git push → EC2에서 deploy.sh 1줄.
git push # 로컬에서 ssh ec2-user@<EIP> "cd tri-pro-poc && ./scripts/deploy.sh"
DB 리셋 후 재배포데이터 삭제
ssh ec2-user@<EIP> "cd tri-pro-poc && ./scripts/deploy.sh --reset"
.env만 변경 후 재기동 (코드 pull X)EC2
ssh ec2-user@<EIP> "cd tri-pro-poc && ./scripts/deploy.sh --no-pull"
