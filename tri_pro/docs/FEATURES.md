# TRI-PRO POC · 기능 안내서

> 의료 전문가가 아니어도 읽을 수 있도록 쓴 기능 설명서.
> 환자·의료진·시스템이 각각 무엇을 하는지, 왜 만들었는지를 그림과 시나리오로 풀어냅니다.

---

## 0. 한 줄 요약

**TRI-PRO**는 항암 치료를 받는 환자가 매일 자신의 컨디션을 짧게 입력하면,
의사가 진료 전 5분 만에 "지난 주 이 환자에게 무슨 일이 있었는지" 한눈에 보는 시스템입니다.

```
환자(앱)에서 "오늘 손발이 많이 저려요" 응답
        ↓
시스템이 자동으로 점수화 + 위험도 평가
        ↓
의사(웹)에서 외래 직전 "C-1042 환자 신경병증 RED 알림" 자동 표시
```

---

## 1. 누구를 위한 것인가

| 사용자 | 무엇을 얻는가 |
|---|---|
| 🧑‍🦰 **환자** | 진료실에서 "그동안 어떠셨어요?"에 잘 떠오르지 않던 증상을 평소에 차근차근 기록 |
| 👴 **고령 환자** | 큰 글씨·얼굴 표정 그림·한 화면 한 질문으로 부담 없이 응답 |
| 👨‍⚕️ **의사** | 외래 5분 전, 그 환자의 1주일이 한 줄 요약 + 추세 그래프로 한눈에 |
| 🏥 **병원 운영팀** | 응답률·이상 신호 환자 현황을 매일 자동 집계 |

## 3. 환자가 보는 화면

### 3.1 홈 — 시연 환자 5명 선택

C-Rehab POC 환경이라 **합성 환자 5명**이 미리 준비돼 있습니다 (실제 환자 X).

```
┌──────────────────────────────────────────────┐
│  TRI-PRO · C-Rehab POC                        │
│  환자앱                                        │
│                                                │
│  ┌──────────────────────────────────────────┐ │
│  │ 이○○  C-1042  🔔 구독중 (1)              │ │
│  │ 73세 · C18.9(결장암) · 사이클 D14         │ │
│  │  [PRO 시작]  [어르신]  [🔔 푸시 구독]    │ │
│  └──────────────────────────────────────────┘ │
│  ┌──────────────────────────────────────────┐ │
│  │ 박○○  C-2103                             │ │
│  │ 58세 · C20(직장암) · D7                   │ │
│  │  ...                                      │ │
│  └──────────────────────────────────────────┘ │
│  (… 총 5명)                                   │
│                                                │
│  🤖 LLM 가드레일 시연 (Free Talk)             │
└──────────────────────────────────────────────┘
```

**무엇을 하는가**
- 환자별로 **PRO 시작·어르신 모드 진입·푸시 구독** 버튼
- "🔔 구독중" 배지로 푸시 받을 준비 상태 표시
- 의료진이 알림을 보내려면 환자가 먼저 "푸시 구독" 한 번 눌러야 함 (브라우저가 알림 권한 요청)

**왜 만들었나**
- 시연용이라 5명 카드를 한 화면에. 실제로는 1인 환자 = 1 계정.
- 푸시 구독 상태가 보여야 의료진이 "어 왜 알림 안 가지?" 같은 혼란이 없음.

### 3.2 PRO 모드 선택 — Full / Quick / No-Change

증상이 매일 비슷한 환자가 매일 똑같이 길게 응답하면 지칩니다. **3가지 모드**로 부담을 분산:

```
┌────────────────────────────────────────────┐
│  오늘의 PRO 모드 선택                          │
│                                             │
│  ┌────────────────────────────────────┐   │
│  │ 🟢 Full 모드                       │   │
│  │ 전체 문항 응답 (5~10분)             │   │
│  │ 모든 항목을 처음부터 답해요.         │   │
│  │ 가장 정확한 데이터가 됩니다.         │   │
│  └────────────────────────────────────┘   │
│  ┌────────────────────────────────────┐   │
│  │ 🟡 Quick 모드                      │   │
│  │ 카테고리 스크리닝 후 양성만 (2~3분)  │   │
│  │ 5개 영역 중 '있음'만 골라 세부로.    │   │
│  └────────────────────────────────────┘   │
│  ┌────────────────────────────────────┐   │
│  │ ⚪ No-Change 모드                  │   │
│  │ 직전 응답 그대로 확인 (15초)        │   │
│  │ 어제와 같다면 한 번에 확인.          │   │
│  └────────────────────────────────────┘   │
└────────────────────────────────────────────┘
```

**무엇을 하는가**
- **Full**: 모든 문항 (PRO-CTCAE 5증상 + HADS 14문항 등) 처음부터 응답. 가장 정확.
- **Quick**: 5개 카테고리(피로·소화·신경·기분·수면) 중 "있음"만 체크 → 양성 카테고리만 세부 문항 진행. 시간 70% 단축.
- **No-Change**: "어제와 같아요" 한 번 누르면 직전 Full 응답 그대로 복제. 15초.

**안전장치**
- **No-Change 30일 만료** — 30일 이상 Full 응답 없으면 자동 비활성화
- **No-Change 연속 3회 상한** — 연속 3회 사용 시 비활성화 → 최소 한 번은 직접 응답
- **월 1회 Full 의무** — 데이터 신뢰도 유지

**왜 만들었나**
임상 시험에서 PRO 응답률이 50% 미만으로 떨어지면 데이터로 못 쓴다고 알려져 있습니다. 환자가 "매일 같은데 왜 또 물어?"라고 느끼면 응답을 안 함. 3단 모드로 **응답률 ↑ + 데이터 신뢰도 유지** 양쪽 잡습니다.

**무엇을 하는가**

**왜 이 도구들인가**
- PRO-CTCAE: 미국 NCI(National Cancer Institute) 공인. 항암 부작용 표준.
- HADS: 가장 널리 쓰이는 우울·불안 평가. 단 5분.

### 3.4 결과 화면 — 즉시 피드백

응답 끝나면 **5초 안에** 그 자리에서 결과를 봅니다.

```
┌──────────────────────────────────────────────┐
│  오늘의 결과                                    │
│                                                │
│  ┌─────────────────────────────────────────┐ │
│  │  🔴 신경병증                             │ │
│  │  많이 저리다 (3점)                       │ │
│  │  최근 7일 연속 진행됨                    │ │
│  └─────────────────────────────────────────┘ │
│                                                │
│  ┌─────────────────────────────────────────┐ │
│  │  🟡 HADS-A 불안                         │ │
│  │  8점 (경계값)                            │ │
│  └─────────────────────────────────────────┘ │
│                                                │
│  📈 7일 추세                                  │
│   [차트 — 신경병증 1→1→2→2→3→3→3]            │
│                                                │
│  주치의에게 다음 외래 시 알림 전송 완료           │
└──────────────────────────────────────────────┘
```

**무엇을 하는가**
- 즉시 채점 (5종 도구 모두 결정론 알고리즘 — 100% 재현 가능)
- 🔴 RED·🟡 YELLOW·🟢 정상 신호등
- 7일 추세 그래프
- 의료진에게 자동 알림 (주치의 어드민에 RED 표시)

**왜 만들었나**
환자에게 응답이 "어디에 쓰이는지" 즉시 보여주지 않으면 동기 부여가 떨어집니다. "내가 입력한 게 의사에게 갔구나"가 보여야 다음번에도 응답.

### 3.5 어르신 모드 — 한 화면 한 질문 + 얼굴 척도

```
┌──────────────────────────────────────────────┐
│  ← 뒤로                          1 / 19       │
│  █░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░             │
│                                                │
│  🩺 몸 상태                                    │
│                                                │
│         피곤함 — 얼마나 심하게 어땠어요?         │
│                                                │
│  ┌────────────────────────────────────────┐  │
│  │  😀  전혀 안 그래요                       │  │
│  │  🙂  조금요                              │  │
│  │  😐  보통이에요                          │  │
│  │  😣  많이 그래요                         │  │
│  │  😭  정말 너무 심해요                     │  │
│  └────────────────────────────────────────┘  │
│                                                │
│  잘 하고 계세요!                              │
└──────────────────────────────────────────────┘
```

**무엇을 하는가**
- **One-Question-One-Screen**: 한 화면에 한 질문만
- **22pt 이상 큰 글씨** + 큰 버튼
- **얼굴 척도** (Wong-Baker faces) — 5단계 표정으로 0~4점 매핑
- HADS는 4단계 얼굴 (전혀/가끔/자주/거의 매일)
- 상단에 보호자/119 원터치 (`tel:` 링크) 항상 노출

**중요 — 모드 등가성**
일반 모드(라디오 버튼)와 어르신 모드(얼굴) **둘 다 같은 0~4 숫자로 저장**됩니다.
즉 같은 "심하게 피곤함" 응답은 모드 무관하게 **같은 점수·같은 위험도**로 처리.
의료진이 "이 환자 일반 모드로 답한 거랑 어르신 모드로 답한 거랑 비교 가능?" 걱정 없음.

**왜 만들었나**
- 항암 환자 다수가 60대 이상. 작은 글씨·복잡한 라디오는 응답 자체를 포기시킴.
- 얼굴 척도는 1968년 Wong이 만든 임상 표준 — 의료진도 신뢰.
- 보호자/119 원터치는 응급 상황 즉시 대응.

### 3.6 LLM Free Talk + 4종 가드레일

환자가 자유롭게 메시지 입력. 의료적으로 위험할 수 있는 질문은 **사전에 차단**.

```
┌──────────────────────────────────────────────┐
│  건강 상담                                     │
│                                                │
│  ⚠️ POC: LLM Mock 모드                       │
│                                                │
│  [🟢 정상]  [🔴 응급]  [🟡 처방]              │
│                                                │
│  ┌──────────────────────────────────┐        │
│  │ (사용자) 손발이 저려요              │        │
│  └──────────────────────────────────┘        │
│  ┌──────────────────────────────────┐        │
│  │ Oxaliplatin·Paclitaxel 계열 항암제는│        │
│  │ 말초신경병증을 유발할 수 있어요.     │        │
│  │ 찬물·추운 환경 노출을 줄이고...     │        │
│  │                                    │        │
│  │ — ASCO Practice Guideline 2024    │        │
│  │ — 이 안내는 일반 정보이며 의료 자문을│        │
│  │   대체하지 않습니다.                │        │
│  └──────────────────────────────────┘        │
│                                                │
│  [메시지 입력] [보내기]                        │
└──────────────────────────────────────────────┘
```

**4종 안전장치 (가드레일)**

| # | 입력 예시 | 시스템 동작 |
|---|---|---|
| 🚨 응급 | "숨이 안 쉬어져요" | 빨간 화면 takeover + 119 전화 큰 버튼 (200ms 이내) |
| 🚫 처방 차단 | "약 더 먹어도 되나요?" | "주치의 상담이 필요해요" 고정 응답 |
| 📚 교육 카드 | "손발이 저려요" | 가이드라인 출처 인용 + 의료자문 면책 |
| 🔒 개인정보 | "내 번호 010-1234-5678" | audit 저장 시 `[전화번호]`로 자동 가림 |

**왜 이렇게까지 차단하나**
- LLM이 "약 1알 더 드세요" 같은 답변을 하면 **의료법 위반 + 환자 위험**
- 응급 상황(호흡곤란 등)에서 LLM이 사담하면 골든타임 놓침
- 환자가 채팅에 주민번호·전화번호를 무심코 적으면 audit log에 평문 저장돼 사고
- 모든 LLM 응답에 출처(ASCO/NCCN 등) 강제 — "어디서 나온 정보인지" 환자도 의료진도 추적 가능

**기술적으로**
- 응급·처방 키워드는 **LLM이 보기 전에 사전 필터** — LLM이 어떻게 답하든 동일하게 차단됨
- 응답 시간 **200ms 이내** (LLM은 보통 2초+이라 사전 필터로 가드레일 안전)
- POC는 **Mock 모드** (LLM 호출 0원). 실제 모델 연동은 같은 가드레일 그대로 적용.

- 5명 환자를 **한 화면**에. 자동 새로고침 10초.
- **3색 신호등**: 🔴 즉시 외래 / 🟡 주의 관찰 / 🟢 안정
- 각 환자에 **One-Line Summary** ("말초신경병증 7일 연속 3점 지속") — 1초 안에 파악

**왜 만들었나**
의사 한 명이 외래 1세션에 환자 30명 진료. 한 명당 5분. 그 5분에 "지난 주 어땠어요?" 묻기엔 너무 짧음. **외래 시작 전에** 위험 환자 미리 거르는 화면이 필요.

**핵심 기능 5가지**

1. **자동 추천**: ICD-10(병명) + 나이로 도구 자동 선택
   - 대장암 + 75세 미만 → PRO-CTCAE·HADS·FACT-C 필수
   - 대장암 + 75세 이상(어르신) → FACT-C는 선택으로 (응답 부담 감소)
   - 폐암 → PRO-CTCAE·HADS 필수, FACIT-F 권장
   - 위암 + 50세 미만 → HADS 주 1회 (정서 모니터 강화)

2. **필수 vs 선택**: 🔒 필수 도구는 잠금 (꼭 측정해야 하는 핵심)

3. **세부 변경**: 자동 추천 후 의사가 환자 특성에 맞게 미세조정
   - 신경병증만 빡세게 보겠다 → 다른 증상 OFF
   - 우울만 보겠다 → HADS-D만, A는 OFF

4. **임계값 조정**: 보수적인 의사는 RED 기준을 1점으로, 공격적인 의사는 3점으로

5. **커스텀 세트 적용**: "FOLFOX 4cycle 추적" 같은 템플릿을 1클릭으로 적용

**왜 만들었나**
같은 결장암이라도 환자가 70대인지 40대인지·신경병증 위험 약물인지에 따라 **모니터링 우선순위가 달라짐**. 모든 환자에게 같은 18문항을 강요하면 응답률 30% 이하로 추락.

**환자 화면 미리보기**
의사가 "이렇게 설정하면 환자가 뭘 보지?" 궁금할 때 **iframe으로 환자앱 화면을 그대로 표시**. 응답 부담 직접 체감.

## 5. 시스템이 자동으로 하는 것

### 5.1 결정론 채점

```
환자 응답: 신경병증 강도 = 3
        ↓
[score_pro_ctcae_item("neuropathy", "severity", 3)]
        ↓
{ symptom: "neuropathy", attribute: "severity",
  score: 3, severity_grade: "severe" }
        ↓
[evaluate_mcid_pro_ctcae([{score:3}], history=[2,2,3,3])]
        ↓
"red"   ← 3점 + 직전 2일 모두 ≥2점
```

**무엇을 보장**
- **동일 입력 → 동일 출력 100%** (재현성)
- LLM은 채점에 일절 관여 X
- 모든 도구의 역채점·결측·합산 룰 명시적 코드

**왜 중요한가**
임상 의사결정 도구는 **재현성**이 의무. "어제는 RED였는데 오늘 같은 응답인데 YELLOW?" 같은 상황은 절대 안 됨.

### 5.2 모드 등가성

```
일반 모드 (라디오 0~4)  ─┐
                       ├─→ raw_value = 3 ─→ scoring engine ─→ ProScore
어르신 모드 (얼굴 0~4)  ─┘                  (모드 무관)
```

같은 raw_value면 **같은 점수, 같은 RED/YELLOW flag**. 일반·어르신 응답 데이터가 한 환자 trend에서 섞여도 일관성 보장.

### 5.3 자동 시드

배포 후 첫 부팅 시 환자 0명이면 자동으로 5명 합성 환자 + 7일 PRO 데이터 생성.
시연 환경 재배포할 때마다 수동 시드 불필요.

## 9. 자주 묻는 질문

**Q. LLM은 어떤 모델을 쓰나요?**
POC는 **Mock 모드** (LLM 호출 0건, 비용 0원). 결정론 템플릿 6종으로 응답.
운영 시 자체 모델 또는 Anthropic API 연동. 가드레일은 동일하게 작동.

**Q. 환자 데이터 어디 저장되나요?**
POC는 SQLite 파일 1개 (서버 1대). 합성 환자 5명. 실제 환자 데이터 0건.
운영 시 RDS Postgres + 자동 백업 + 다중 AZ.

**Q. 의료기기 인증 받았나요?**
아닙니다. POC는 **임상 의사결정 보조 도구 검증용** 단계.
PRO 점수는 보조 시그널이며, 최종 진단·처방은 항상 의사가 함.

**Q. 푸시 알림이 의사 휴대폰으로 가나요?**
POC는 **환자 휴대폰 → 환자 본인 알림**까지만. 의사용 모바일 푸시는 운영 단계에 추가.

**Q. 환자 본인 인증은?**
POC는 미적용 (시연용). 운영 시 휴대폰 OTP + 본인인증 필수.

**Q. 다른 암종도 지원하나요?**
현재 자동 추천 5종 — 대장암(C18~21) / 폐암(C34) / 위암(C16) / 유방암(C50) / 기타.
PRO 도구는 5개라 어떤 암종도 사용 가능. 자동 추천만 추가 코드.
