C. 결정론 채점 알고리즘 FN-PRO-006

원칙: 동일 입력 → 동일 출력 100%. LLM 미관여. 모든 역채점·결측·합산 룰을 명시 코드로 구현.

PRO-CTCAE 채점

# 항목별 채점 — 합산하지 않고 분포 보존 score_pro_ctcae_item(symptom, attribute, raw): return { "symptom": symptom, "attribute": attribute, "score": raw, # 0..4 "severity_grade": ["none","mild", "moderate","severe","very_severe"][raw], } # Composite — max(강도, 일상) — alert용 score_pro_ctcae_composite(items): relevant = [i.score for i in items if i.attribute in ("severity", "interference")] return {"composite": max(relevant)}

HADS 채점

# 14문항 → A·D 분리 + 역채점 7개 적용 HADS_REVERSE = {0:3, 1:2, 2:1, 3:0} HADS_ITEMS_REVERSE = ["D1","D2","D3","A4","D5","D6","D7"] score_hads_subscale(responses, sub): total = 0 for code, reversed_ in items_of(sub): raw = responses[code] adjusted = HADS_REVERSE[raw] if reversed_ else raw total += adjusted classification = ( "normal" if total ≤ 7 else "borderline" if total ≤ 10 else "case" ) return {"value": total, "classification": classification}

FACT-C / FACIT-F / PSQI

  • FACT-C: 5 서브스케일(PWB 7·SWB 7·EWB 6·FWB 7·CCS 9) → 결측 50% 이상 시 평균 보정 → TOI = PWB+FWB+CCS, FACT-G = PWB+SWB+EWB+FWB
  • FACIT-F: 13문항 합 (0..52). 점수 ≤30 = severe, 31~43 = moderate, ≥44 = normal. MCID 3점.
  • PSQI: 7 컴포넌트 합 (0..21). ≥5 = poor sleep, ≥10 = severe.

D. MCID·Alert 정책

PRO-CTCAE — 지속성 기반

evaluate_mcid_pro_ctcae(scores, history, red_threshold=2, persist_days=2): today_max = max(s.score for s in scores) # 직전 N일 모두 ≥ 임계값 → RED 확정 if today_max ≥ red_threshold and len(history) ≥ persist_days and all(h ≥ red_threshold for h in history[-persist_days:]): return "red" if today_max ≥ red_threshold: return "yellow" return None

E. Pre-Visit Report 의사결정 분기 FN-RPT-008·010

One-Line Summary 자동 생성 룰

if len(red_alerts) ≥ 2: return "{n}개 영역 RED — 즉시 외래 또는 전화 진료 권고." elif red_alert is HADS: return "HADS-{sub} {value}점 — 임상적 이상. 전문 상담 의뢰 검토." elif red_alert is PRO-CTCAE: sym = red_alert.subscale # neuropathy/fatigue/nausea/appetite/diarrhea return 도구별 전용 템플릿 (예: "말초신경병증 7일 연속 3점 지속.") elif yellow_alerts: return "HADS-{sub} {value}점 진입 (경계값). 정서적 지지 권장." else: return "지난 {days}일간 PRO 점수 안정적. 특이 변화 없음."

의사결정 우선순위 트리

┌─ Pre-Visit Report 진입
│
├─ One-Line Summary 1초 파악
│   ├─ "다중 영역 RED" → 외래·전화 즉시
│   ├─ "신경병증 N일 연속" → 신경독성 평가 / 용량 조정 검토
│   ├─ "HADS 11점 case" → 정신과 협진
│   └─ "안정적" → 통상 진료
│
├─ Key Changes 카드로 RED/YELLOW 도구·점수 확인
├─ 7일 추세 차트 (PRO-CTCAE neuropathy + HADS-A 오버레이)
├─ 라이프로그 대조 (활동량·체중·식이 — Pearson r)
├─ 이수율 (응답 횟수 / 예상 + 미응답 일수)
│
└─ 일별 PRO 응답 상세 클릭 → 환자 발화·원 점수 추적
중요: One-Line Summary는 LLM이 만들지 않습니다. 결정론 템플릿 9종 매칭으로 100% 재현 가능. LLM 리라이트는 Phase 1+에서 옵션으로 추가 예정 (audit 의무).

H. LLM 가드레일 FN-LLM-005~011

분기 우선순위 (Top-down)

1. PII 마스킹 (FN-LLM-011)
   - 휴대폰·주민번호·이메일·이름 자동 redact
   - audit log·LLM 호출 모두 redacted text만 사용
        ↓
2. 응급 키워드 12개 (FN-LLM-008)
   - "숨이 안 쉬어져요", "쓰러질 것 같아요" 등 사전 매칭
   - 매칭 시 즉시 takeover (SLA < 200ms)
   - 119/응급실 전화 큰 버튼 + 보호자 호출
        ↓
3. Out-of-Scope 패턴 20개 (FN-LLM-007)
   - "약 더 먹어도?", "용량 늘려도?" 등 진단·처방·복용량
   - 고정 응답: "주치의 상담이 필요해요"
   - 100% 차단 (POC KPI 검증)
        ↓
4. Education 카드 (FN-LLM-006)
   - "손발이 저려요" → 신경병증 관리 카드 (ASCO Guideline)
   - "피곤해요" → 암성 피로 카드 (NCCN Guideline)
   - 모든 카드에 source citation 자동 (FN-LLM-009)
        ↓
5. 정상 발화 (FN-LLM-005)
   - Mock 모드: 결정론 템플릿 6종 (deterministic 선택)
   - Real 모드: Anthropic API 호출 (가드레일 통과 후)
   - 모든 응답에 의료자문 면책 footer 자동 부착

변경 이력 가시화

JSON diff을 사람이 읽을 수 있는 이벤트로 자동 변환:

  • ✅ 활성화 / ❌ 비활성화
  • ➕ 추가 / ➖ 제거
  • 🔄 빈도 변경 (예: 매일 → 주 1회)
  • 🔧 임계값 변경 (예: HADS RED 11 → 8)
  • 📚 ICD-10 기본값 적용
  • 📋 커스텀 세트 적용

재현성 보장

  1. 모든 채점은 결정론 알고리즘 — 같은 입력·같은 임계값 = 같은 결과
  2. 임계값 변경 시점이 audit으로 기록되어 "그 시점 어떤 임계값으로 어떤 결과가 나왔나" 추적 가능
  3. ProResponse·ProScore는 append-only — 수정·삭제 불가
  4. LLM 응답은 Mock 모드면 deterministic, Real 모드여도 prompt + response 모두 audit

  • PSQI 채점 알고리즘 간소화 버전 (C4 컴포넌트 시간 비율 계산 생략)

MVP 전환 시 임상 검토 필요 영역

  1. MCID 임계값 confirmation — 도구별 한국 항암 환자 데이터 기반 검증 필요 (현재 미국 기준 적용)
  2. FACT-C 한국어 번역 — 임상 검증된 한국어판 도입 (한국어 FACT-C 번역본 존재)
  3. 응급 키워드 확장 — 12개 → 의료진 자문으로 50개 이상
  4. Out-of-Scope 패턴 — 한국 환자 발화 데이터 수집 후 패턴 보강
  5. One-Line Summary 템플릿 — 의사 자문으로 9개 → 30개 이상 (도구·증상 조합)
  6. 라이프로그 상관 분석 — 현재 더미 텍스트, MVP는 활동량 API 연동
