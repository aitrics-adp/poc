의료 전문가가 아니어도 읽을 수 있도록 쓴 기능 설명서. 환자·의료진·시스템이 각각 무엇을 하는지, 왜 만들었는지를 그림과 시나리오로 풀어냅니다.

0. 한 줄 요약

TRI-PRO는 항암 치료를 받는 환자가 매일 자신의 컨디션을 짧게 입력하면, 의사가 진료 전 5분 만에 "지난 주 이 환자에게 무슨 일이 있었는지" 한눈에 보는 시스템입니다.

환자(앱)에서 "오늘 손발이 많이 저려요" 응답
        ↓
시스템이 자동으로 점수화 + 위험도 평가
        ↓
의사(웹)에서 외래 직전 "C-1042 환자 신경병증 RED 알림" 자동 표시

3. 환자가 보는 화면

3.1 홈 — 시연 환자 5명 선택

C-Rehab POC 환경이라 합성 환자 5명이 미리 준비돼 있습니다 (실제 환자 X).

각 환자별 PRO 시작·어르신 모드 진입·푸시 구독 버튼. "🔔 구독중" 배지로 푸시 받을 준비 상태 표시. 의료진이 알림을 보내려면 환자가 먼저 "푸시 구독"을 한 번 눌러야 합니다.

3.2 PRO 모드 선택 — Full / Quick / No-Change

증상이 매일 비슷한 환자가 매일 똑같이 길게 응답하면 지칩니다. 3가지 모드로 부담을 분산:

모드소요용도
Full5~10분모든 문항 처음부터 응답. 가장 정확.
Quick2~3분5개 카테고리 스크리닝 후 양성만 세부 진행.
No-Change15초"어제와 같아요" 한 번 누르면 직전 응답 복제.

안전장치: No-Change 30일 만료 / 연속 3회 상한 / 월 1회 Full 의무 — 데이터 신뢰도 유지.

임상 시험에서 PRO 응답률이 50% 미만으로 떨어지면 데이터로 못 씁니다. 환자가 "매일 같은데 왜 또 물어?"라고 느끼면 응답을 안 함. 3단 모드로 응답률↑ + 데이터 신뢰도 유지 양쪽 잡습니다.

3.3 PRO 응답 (Full 모드)

  • PRO-CTCAE (NCI 표준): 항암 부작용 5증상 — 피로·식욕부진·오심·설사·신경병증. 빈도·강도·일상 방해 정도 측정
  • HADS (병원 불안·우울 척도): 14문항으로 정신건강 평가
  • 진행률 막대로 "몇 개 남았는지" 명확히

3.4 결과 화면 — 즉시 피드백

응답 끝나면 5초 안에 그 자리에서 결과를 봅니다. RED YELLOW 정상 신호등 + 7일 추세 그래프 + 의료진 자동 알림.

환자에게 응답이 "어디에 쓰이는지" 즉시 보여주지 않으면 동기 부여가 떨어집니다. "내가 입력한 게 의사에게 갔구나"가 보여야 다음번에도 응답.

3.5 어르신 모드 — 한 화면 한 질문 + 얼굴 척도

  • One-Question-One-Screen: 한 화면에 한 질문만
  • 22pt 이상 큰 글씨 + 큰 버튼
  • 얼굴 척도 (Wong-Baker faces) — 5단계 표정으로 0~4점 매핑
  • HADS는 4단계 얼굴 (전혀/가끔/자주/거의 매일)
  • 상단에 보호자/119 원터치 항상 노출
중요 — 모드 등가성
일반 모드(라디오)와 어르신 모드(얼굴) 둘 다 같은 0~4 숫자로 저장됩니다. 같은 "심하게 피곤함" 응답은 모드 무관하게 같은 점수·같은 위험도로 처리. 의료진이 모드별 데이터 비교 가능.

3.6 LLM Free Talk + 4종 가드레일

입력 예시시스템 동작
🚨 "숨이 안 쉬어져요"빨간 화면 takeover + 119 전화 큰 버튼 (200ms 이내)
🚫 "약 더 먹어도 되나요?""주치의 상담이 필요해요" 고정 응답
📚 "손발이 저려요"가이드라인 출처 인용 + 의료자문 면책
🔒 "내 번호 010-1234-5678"audit 저장 시 [전화번호]로 자동 가림
LLM이 "약 1알 더 드세요" 같은 답변을 하면 의료법 위반 + 환자 위험. 응급·처방 키워드는 LLM이 보기 전에 사전 필터로 차단. POC는 Mock 모드 (LLM 호출 0원).

4. 의료진이 보는 화면 (어드민)

4.1 모니터링 대시보드 — 한눈에 5명

  • 5명 환자를 한 화면에. 자동 새로고침 10초.
  • 3색 신호등: RED 즉시 외래 YELLOW 주의 관찰 안정
  • 각 환자에 One-Line Summary ("말초신경병증 7일 연속 3점 지속") — 1초 안에 파악

4.2 Pre-Visit Report — 외래 5분 전 의사결정 페이지

  • One-Line Summary: 의사에게 전달되는 결론 한 줄
  • 7일 추세 차트: 신경병증 + HADS-A 오버레이
  • Key Changes: RED/YELLOW 알림만 모은 카드
  • 라이프로그 대조: 활동량·체중·식이 변화 상관
  • 이수율: 응답 횟수 / 예상 횟수 + 미응답 일수
  • 환자에게 푸시 발송: 한 클릭으로 알림
One-Line Summary 작동 원리: LLM이 만드는 게 아니라 결정론 템플릿 9종으로 만듭니다. RED ≥ 2개면 "다중 영역 RED — 즉시 외래 권고", 신경병증 RED는 "말초신경병증 N일 연속 K점 지속" 등. 100% 재현 가능한 일관된 요약.

4.3 일별 PRO 응답 상세

"이 환자가 정확히 무엇을 어떻게 답했지?" 추적용. 날짜별 카드 펼치면 그 날의 모든 응답 (응답값 + 한국어 라벨 + 원 질문). 채점 결과·MCID flag·세션 모드·일반/어르신 표시 모두 포함.

4.4 PRO 도구 라이브러리 + 상세 페이지

  • 5종 표준 도구(PRO-CTCAE / HADS / FACT-C / FACIT-F / PSQI) 메타정보 + 실제 문항 미리보기
  • 의사가 "이 도구를 환자에게 적용하면 어떤 질문이 갈까?" 직접 확인
  • 커스텀 PRO 세트 — 의사가 자주 쓰는 도구 조합을 템플릿으로 저장

4.5 환자별 PRO 설정 (커스터마이징)

가장 중요한 화면 중 하나 — 환자마다 다른 도구·빈도·임계값 설정.

  1. 자동 추천: ICD-10(병명) + 나이로 도구 자동 선택
    • 대장암 + 75세 미만 → PRO-CTCAE·HADS·FACT-C 필수
    • 대장암 + 75세 이상 → FACT-C는 선택으로 (응답 부담 감소)
    • 폐암 → PRO-CTCAE·HADS 필수, FACIT-F 권장
    • 위암 + 50세 미만 → HADS 주 1회 (정서 모니터 강화)
  2. 필수 vs 선택: 🔒 필수 도구는 잠금
  3. 세부 변경: 자동 추천 후 환자 특성에 맞게 미세조정
  4. 임계값 조정: 보수적 의사는 RED 기준 1점, 공격적 의사는 3점
  5. 커스텀 세트 적용: "FOLFOX 4cycle 추적" 같은 템플릿을 1클릭

4.6 변경 이력 — 가시화 Audit

모든 PRO 설정 변경을 자동 기록. 사람이 읽을 수 있는 이벤트로 변환:

  • ✅ 활성화 / ❌ 비활성화
  • ➕ 추가 / ➖ 제거
  • 🔄 빈도 변경 (예: 매일 → 주 1회)
  • 🔧 임계값 변경 (예: HADS RED 11 → 8)
  • 📚 ICD-10 기본값 적용
  • 📋 커스텀 세트 적용
임상 자료는 변경 추적이 의무 (식약처·임상시험심사). "이 환자 4월 26일 신경병증 데이터를 어떤 임계값으로 계산했지?" 답할 수 있어야 함.

5. 시스템이 자동으로 하는 것

5.1 결정론 채점

환자 응답: 신경병증 강도 = 3
        ↓
score_pro_ctcae_item("neuropathy", "severity", 3)
        ↓
{ symptom: "neuropathy", attribute: "severity",
  score: 3, severity_grade: "severe" }
        ↓
evaluate_mcid_pro_ctcae([{score:3}], history=[2,2,3,3])
        ↓
"red"   ← 3점 + 직전 2일 모두 ≥2점
  • 동일 입력 → 동일 출력 100% (재현성)
  • LLM은 채점에 일절 관여 X
  • 모든 도구의 역채점·결측·합산 룰 명시적 코드

5.2 모드 등가성

일반 모드 (라디오 0~4)  ─┐
                       ├─→ raw_value = 3 ─→ scoring engine ─→ ProScore
어르신 모드 (얼굴 0~4)  ─┘                  (모드 무관)

5.3 자동 시드

배포 후 첫 부팅 시 환자 0명이면 자동으로 5명 합성 환자 + 7일 PRO 데이터 생성. 시연 환경 재배포할 때마다 수동 시드 불필요.

7. 데이터 신뢰성

  1. 표준 도구 사용: PRO-CTCAE(NCI), HADS(공개 표준), FACT-C(FACIT.org) — 모두 임상 검증
  2. 결정론 채점: 동일 입력 → 동일 출력. 100% 재현 가능
  3. 변경 이력 영구 보관: 어떤 임계값으로 어떤 결과가 나왔는지 추적
  4. 테스트 90개 통과: 채점·가드레일·모드 등가성 모두 자동
  5. LLM 의사결정 미관여: 점수 산출·위험 판정은 결정론 코드만

8. 시연 시나리오 (5분)

  1. 의료진 어드민 → 5명 환자 카드 (4명 RED) → C-1042 Pre-Visit 클릭
  2. Pre-Visit Report → "신경병증 7일 연속 3점" 한 줄 → 차트 + Key Changes
  3. 일별 응답 상세 → 4월 28일 펼치기 → 신경병증 3점 + HADS-A 8점 응답
  4. PRO 설정 → C-3027(폐암) → "ICD-10 기본값 불러오기" → FACIT-F 자동 ON
  5. 환자앱 → C-1042 어르신 모드 → 큰 글씨 + 얼굴 척도
  6. LLM Free Talk → "손발 저려요" 출처 카드 / "약 더?" 차단 / "숨이 안 쉬어져요" 119
  7. Cron Jobs → 수동 실행 → 5명 사전계산 결과

각 단계 화면 자동 시연: ./demo/run_demo.sh

9. 자주 묻는 질문

Q. LLM은 어떤 모델을 쓰나요?
POC는 Mock 모드 (LLM 호출 0건, 비용 0원). 결정론 템플릿 6종으로 응답. 운영 시 자체 모델 또는 Anthropic API 연동. 가드레일은 동일하게 작동.

Q. 환자 데이터 어디 저장되나요?
POC는 SQLite 파일 1개 (서버 1대). 합성 환자 5명. 실제 환자 데이터 0건. 운영 시 RDS Postgres + 자동 백업 + 다중 AZ.

Q. 의료기기 인증 받았나요?
아닙니다. POC는 임상 의사결정 보조 도구 검증용 단계. PRO 점수는 보조 시그널이며, 최종 진단·처방은 항상 의사가 함.

Q. 푸시 알림이 의사 휴대폰으로 가나요?
POC는 환자 휴대폰 → 환자 본인 알림까지만. 의사용 모바일 푸시는 운영 단계에 추가.

Q. 환자 본인 인증은?
POC는 미적용 (시연용). 운영 시 휴대폰 OTP + 본인인증 필수.
